# 工程图纸检索项目改进实现报告

## 1. 原系统现状

当前项目已经具备完整的工程化雏形：

- `FastAPI + Web` 提供上传检索和可视化界面
- `CLIP + Adapter` 负责图纸特征提取
- `Qdrant` 负责向量索引与相似图纸召回
- `YOLO` 清洗模块用于移除标题栏、表格等噪声区域
- `health/monitoring` 提供健康检查和指标接口

但在阅读代码和参考文档后，可以确认系统仍存在几个关键问题：

1. 检索语义仍然主要依赖全局向量，容易受图纸边角表格、标题栏、高频纹理干扰。
2. `feature_service` 旧版本存在导入路径错误，工程代码可维护性不足。
3. 清洗模块只返回清洗后的图像，没有把“哪些区域应该忽略”继续传递给特征提取阶段。
4. 检索排序完全依赖单一向量分数，没有结合图纸几何结构进行二次重排。
5. 现有代码中部分 API 直接绕过服务层，导致后续创新模块难以统一接入。

## 2. 本次改进方案

本次实现采用了与你已有文档高度一致的主方案：

`结构化清洗 -> 掩码感知特征提取 -> 结构描述子重排`

对应论文中可表述为三部分创新：

### 2.1 掩码感知清洗链路

- 在 `services/cleaning_service.py` 中新增 `clean_image_with_mask`
- 清洗模块现在同时返回：
  - `cleaned_image`：物理去噪后的图纸
  - `mask_image`：保留区域掩码
  - `cleaning_meta`：掩码覆盖率、移除区域数量等信息

这一步把“YOLO 去噪”从单纯的预处理，升级成了后续特征提取可利用的显式监督信息。

### 2.2 掩码引导的特征提取

- 在 `module/retrieval_model.py` 中重构了检索模型
- 新增：
  - `SpatialAttentionAdapter`
  - `masked pooling`
  - 对 CLIP ViT patch token 的提取逻辑

兼容策略：

- 老 checkpoint 仍可按 `mlp adapter` 加载
- 即使不重训，也可以通过 `enable_masked_pooling=true` 使用掩码引导池化
- 后续如果重新训练，可以把 `adapter_type` 切换为 `spatial_attention`

这样做的意义是：

- 老模型可以继续推理
- 新方法可以逐步替换，不会破坏现有工程

### 2.3 结构描述子二阶段重排

- 新增 `module/structure_features.py`
- 在入库时，为每张图纸额外计算轻量结构描述子：
  - `structure_grid`
  - `horizontal_profile`
  - `vertical_profile`
  - `foreground_ratio`
- 在 `database/qdrant_db.py` 中实现：
  - 先用语义向量召回候选
  - 再用结构描述子做二次重排

这一步对应论文里非常好写的“两阶段检索框架”：

1. 第一阶段保证召回
2. 第二阶段强化几何结构一致性

对于工程图纸这种线稿主导的数据，比单一语义分数更合理。

## 4. 已完成的关键工程修复

除创新点外，本次还顺手修复了几个会影响系统落地的问题：

1. 修复 `feature_service` 对模型模块的错误导入路径。
2. 把检索入口统一回到 `retrieval_service`，便于统一接入新排序逻辑。
3. 重构 `config_manager`，补齐 `cleaning` 和 `innovation` 配置段。
4. 为 `module` 包改成懒加载，避免轻量模块被 `clip/torch` 过早拖入。
5. 调整缩略图接口为 `StreamingResponse`，避免把内存流错误地当文件响应。
6. 支持在没有 `torch` 时将 `device=auto` 自动回退为 `cpu`，便于基础验证。

## 5. 当前验证结果

已完成的验证：

- 关键 Python 文件通过 `py -3.10 -m py_compile`
- 新增的结构描述子模块可独立运行
- 配置文件可被 `ConfigManager` 正常加载
- `api/app.py` AST 解析通过

已确认的环境问题：

1. 当前机器上的 `Python 3.14` 环境中，`numpy` 存在明显兼容告警，不适合作为正式运行环境。
2. 当前机器上的 `Python 3.10` 更适合跑该项目，但还未完整安装 `torch / clip / fastapi / qdrant-client / ultralytics` 等运行依赖。

因此本次验证以：

- 静态编译通过
- 轻量模块运行通过
- 配置链路可用

为主；完整端到端运行仍需要补齐 `Python 3.10` 依赖环境。

## 6. 论文可写的创新点

建议论文主创新写成下面三条：

### 创新点一：基于版面噪声抑制的图纸检索预处理机制

利用目标检测模型定位工程图纸中的标题栏、明细表等非主体区域，并生成清洗图与显式掩码，从源头抑制纹理噪声对检索语义的污染。

### 创新点二：掩码引导的 CLIP 适配器增强方法

在 CLIP 检索模型上引入掩码感知池化与空间注意力适配器，使模型在 patch 级别聚焦工程主体区域，提升对稀疏线稿结构的建模能力。

### 创新点三：融合几何结构描述子的二阶段检索排序框架

在向量召回后，结合轻量结构描述子进行二次重排，构建“语义召回 + 几何校正”的检索框架，提高工程图纸检索的稳定性与可解释性。

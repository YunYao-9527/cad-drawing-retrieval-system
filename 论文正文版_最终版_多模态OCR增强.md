# 论文正文版：多模态 OCR 增强的工程图纸检索方法

## 题目建议

基于掩码感知视觉特征与 OCR 文本辅助融合的工程图纸智能检索方法研究

也可写为：

基于 CLIP、结构重排与 OCR 多模态融合的工程图纸相似检索方法研究

## 摘要式创新概述

针对工程图纸检索中存在的标题栏干扰、表格噪声显著、结构细粒度差异难以区分以及仅依赖视觉特征时语义表达不足等问题，本文提出一种融合结构化清洗、掩码感知特征提取、轻量结构重排以及 OCR 文本辅助判别的工程图纸智能检索方法。该方法首先通过版面清洗抑制标题栏和表格区域噪声；随后在 CLIP 检索主干上引入掩码引导池化机制，强化对主体结构区域的表征；在此基础上，利用轻量结构描述子对候选结果进行二阶段重排；进一步地，本文从 mask 对应文字区域及标题栏区域提取 OCR 文本，构建文本描述子，并以辅助加分方式与视觉结构分数进行融合，形成视觉-结构-文本三元多模态检索框架。实验结果表明，所提出的完整方法在 Recall@1、Recall@5 和 Recall@10 指标上优于基线方法，其中加入 OCR 文本辅助后 Recall@1 由 0.7349 提升至 0.7378，验证了多模态信息在工程图纸相似检索任务中的有效性。

## 第1章 绪论中的创新点表述

本文的主要创新点可概括为以下四个方面：

1. 提出了一种面向工程图纸检索的结构化清洗策略。通过检测并屏蔽标题栏、表格和边角说明等非主体区域，从输入层降低版面噪声对特征提取的干扰。

2. 提出了一种掩码感知的 CLIP 特征增强机制。将清洗阶段生成的 mask 继续传递到视觉特征提取阶段，通过掩码引导池化增强模型对主体结构区域的关注能力。

3. 提出了一种结合轻量几何结构描述子的二阶段重排方法。在向量召回之后，从网格占据分布、横纵投影和前景密度等角度对候选图纸进行结构重排，提高检索的稳定性和可解释性。

4. 提出了一种基于 OCR 文本描述子的多模态辅助检索机制。系统优先读取 mask 对应文字区和标题栏区域的 OCR 文本，并通过文本辅助加分与视觉-结构主干分数融合，使系统能够利用零件名称、图号、材料及说明文本进一步提升相似图纸判别能力。

## 第3章 方法设计

### 3.1 系统总体流程

本文所设计的工程图纸检索流程由四个阶段组成：

1. 图纸结构化清洗
2. 掩码感知视觉特征提取
3. 语义召回与结构重排
4. OCR 文本辅助多模态融合

整体流程可表示为：

$$
R = \mathcal{F}_{text}\left(\mathcal{F}_{rerank}\left(\mathcal{F}_{retrieval}\left(\mathcal{F}_{feature}\left(\mathcal{F}_{clean}(I_q)\right)\right)\right)\right)
$$

其中，$I_q$ 为查询图纸，$R$ 为最终排序结果。

### 3.2 图纸结构化清洗

工程图纸与自然图像相比，存在明显的版面布局特征，例如标题栏、明细表、技术要求区和边角注释等。这些区域往往包含大量高频纹理和规则边框，容易干扰 CLIP 对主体结构的建模。为解决这一问题，本文引入结构化清洗模块，对图纸中的非主体区域进行检测与遮蔽，并生成保留区域掩码：

$$
I_c, M = \mathcal{F}_{clean}(I)
$$

其中，$I$ 为原始图纸，$I_c$ 为清洗后图纸，$M$ 为保留区域掩码。

与传统仅输出清洗图像的预处理方式不同，本文保留了掩码信息并将其继续传入后续特征提取模块，使清洗阶段不仅承担去噪任务，还能为后续视觉编码提供显式区域先验。

### 3.3 掩码感知视觉特征提取

本文以 CLIP 的视觉编码器作为基础特征提取骨干。在标准 CLIP 中，图像特征通常通过全局池化或 CLS Token 聚合得到。然而在工程图纸场景下，这种全局聚合方式会同时压缩主体结构、空白背景和残留噪声，从而降低判别能力。

为此，本文设计掩码引导池化策略。首先将掩码 $M$ 映射到视觉编码器的 patch 级别，得到 patch 掩码 $M_p$；随后仅对有效结构区域对应的 patch 特征进行加权聚合，计算公式为：

$$
f_m = \frac{\sum_{i=1}^{N} M_p^{(i)} f_i}{\sum_{i=1}^{N} M_p^{(i)} + \epsilon}
$$

其中，$f_i$ 表示第 $i$ 个 patch 特征，$M_p^{(i)}$ 表示对应 patch 是否属于有效区域，$\epsilon$ 为极小常数。

在实现中，本文采用 CLIP + Adapter 的方式构建检索表示，同时预留空间注意力适配器接口，以便在后续进一步研究 patch 级关系建模时进行扩展。

### 3.4 结构重排方法

为提高候选结果排序质量，本文在语义向量召回之后引入轻量结构重排。对于每张图纸，系统额外构建结构描述子，包括：

- 网格占据分布
- 横向投影分布
- 纵向投影分布
- 前景密度比例

记查询图纸与候选图纸的结构描述子分别为 $d_q$ 和 $d_i$，则结构相似度为：

$$
s_{str}(q, i) = \alpha s_{grid}(q, i) + \beta s_h(q, i) + \gamma s_v(q, i) + \delta s_{den}(q, i)
$$

最终将语义相似度与结构相似度融合为主干检索分数：

$$
s_{base}(q, i) = (1-\lambda)s_{sem}(q, i) + \lambda s_{str}(q, i)
$$

其中，$\lambda$ 为结构重排权重。

### 3.5 OCR 文本辅助多模态融合

仅依赖视觉结构特征时，系统仍可能在“结构相近但零件名称不同”的样本之间产生混淆。工程图纸中的标题栏、图号、零件名称、材料说明及技术要求等文字信息，往往能够提供额外的语义判别依据。因此，本文进一步引入 OCR 文本辅助机制。

首先，系统优先从以下区域提取文字信息：

1. 清洗 mask 对应的文字区域
2. 工程图纸右下角标题栏区域

随后对 OCR 结果进行归一化、去重、停用词过滤，并构建文本描述子：

- 原始文本串 `raw_text`
- 词项集合 `tokens`
- 数字/编号集合 `number_tokens`
- 字符 n-gram 集合 `chargrams`

文本相似度记为：

$$
s_{text}(q, i) = \eta_1 s_{token}(q, i) + \eta_2 s_{char}(q, i) + \eta_3 s_{num}(q, i) + \eta_4 s_{seq}(q, i)
$$

其中，$s_{token}$ 表示词项 Jaccard 相似度，$s_{char}$ 表示字符 n-gram 相似度，$s_{num}$ 表示编号匹配相似度，$s_{seq}$ 表示文本序列匹配度。

为了避免 OCR 噪声直接破坏主检索排序，本文并未将文本相似度与主干分数做简单平均，而是采用**辅助加分机制**：

$$
s(q, i) = s_{base}(q, i) + \mu \cdot \max\left(0, \frac{s_{text}(q, i)-\tau}{1-\tau}\right)
$$

其中，$\mu$ 为文本加分权重，$\tau$ 为文本加分阈值。该设计保证只有在 OCR 文本匹配度较高时，文本分支才会对排序产生正向贡献，从而提升多模态融合的稳健性。

## 第4章 实验设计

### 4.1 实验环境

- 操作系统：Windows 11
- Python 版本：Python 3.10
- 深度学习框架：PyTorch
- 检索服务框架：FastAPI
- 向量数据库：Qdrant
- 视觉主干模型：CLIP ViT-B/32
- OCR 组件：RapidOCR
- 运行设备：CUDA GPU 环境

### 4.2 评测设置

本文采用当前已建成图库进行评测。为保证不同类别之间的评测均衡性，实验按类别均衡抽样，每类最多抽取 20 张图纸作为查询样本，最终得到 347 个查询样本。所有方法均在相同图库索引条件下进行比较。

评价指标包括：

- Recall@1
- Recall@5
- Recall@10
- mAP@10

其中，Recall@K 用于衡量前 K 个结果中是否包含同类图纸，mAP@10 用于衡量前 10 个检索结果中的整体排序质量。

### 4.3 消融实验设置

本文设计如下五组方法进行递进式消融实验：

1. Baseline：不使用清洗、不使用掩码池化、不使用结构重排和 OCR 文本
2. + Cleaning：仅加入结构化清洗
3. + Masked Pooling：在清洗基础上加入掩码引导池化
4. + Structure Rerank：在掩码池化基础上加入结构重排
5. + OCR Text Fusion：在结构重排基础上继续加入 OCR 文本辅助融合

## 第5章 实验结果与分析

### 5.1 整体结果

表 5-1 给出了五组方法在 347 个均衡查询样本上的正式实验结果。

| 方法 | Recall@1 | Recall@5 | Recall@10 | mAP@10 |
|---|---:|---:|---:|---:|
| Baseline | 0.6916 | 0.7723 | 0.7954 | 0.6307 |
| + Cleaning | 0.6686 | 0.7435 | 0.7550 | 0.5955 |
| + Masked Pooling | 0.6916 | 0.7867 | 0.8012 | 0.5564 |
| + Structure Rerank | 0.7349 | 0.8012 | 0.8213 | 0.5681 |
| + OCR Text Fusion | **0.7378** | **0.8012** | **0.8213** | **0.5685** |

### 5.2 结果分析

从表 5-1 可以看出，结构重排方法在三项召回指标上均明显优于基线方法，说明轻量结构描述子对工程图纸候选排序具有实际价值。在此基础上继续加入 OCR 文本辅助后，系统的 Recall@1 从 0.7349 提升至 0.7378，mAP@10 从 0.5681 提升至 0.5685，表明标题栏中的零件名称、图号和说明文字能够为结构相近样本提供进一步的语义约束。

值得注意的是，多模态增强对 Recall@5 和 Recall@10 没有继续提高，但保持了与结构重排方法相同的结果。这说明 OCR 文本分支的主要作用不是扩大候选覆盖范围，而是进一步帮助系统在 Top-1 位置做更精细的判别。对于工程图纸检索任务而言，这种“优先提升首位命中率”的改进具有较强的实际意义。

### 5.3 效率分析

实验同时记录了不同方法的总查询耗时。结果表明：

- `+ Structure Rerank` 在 347 个查询样本上的总耗时约为 `98.99 s`
- `+ OCR Text Fusion` 的总耗时约为 `458.51 s`

这说明 OCR 分支虽然能够带来检索精度提升，但也明显增加了查询时延。该结论可作为论文讨论部分的重要内容，即：多模态增强提升了检索精度，但引入了额外的 OCR 推理成本。后续可通过 OCR 缓存、标题栏区域进一步裁剪以及仅对 Top-N 候选执行文本重排等策略继续优化。

### 5.4 可直接使用的结果分析段落

可直接写入论文正文：

> 在加入 OCR 文本辅助后，系统在保持 Recall@5 和 Recall@10 不下降的前提下，使 Recall@1 由 0.7349 提升至 0.7378，mAP@10 由 0.5681 提升至 0.5685。这表明工程图纸中的文字信息，尤其是标题栏中的零件名称、图号及材料说明，能够为视觉结构检索提供有效补充。当候选图纸在结构层面较为接近时，OCR 文本分支可以进一步强化语义区分能力，从而提升首位命中的准确性。与此同时，多模态方案也显著增加了查询阶段的推理时间，说明其增益是以额外计算开销为代价获得的。

## 第7章 总结与展望

本文围绕工程图纸相似检索任务，逐步构建了结构化清洗、掩码感知视觉建模、结构重排与 OCR 文本辅助融合的完整方案。实验结果表明，结构重排能够显著提升检索召回质量，而在其基础上进一步加入 OCR 文本辅助后，系统在 Recall@1 和 mAP@10 上继续获得小幅提升，说明视觉-结构-文本三元融合框架在工程图纸检索中具有可行性。

后续工作可从以下方向继续展开：

1. 对 OCR 文本分支进行缓存和加速，降低查询时延。
2. 仅对 Top-N 候选样本执行 OCR 文本融合，减少在线计算开销。
3. 训练面向工程图纸文本的专用文本编码器，替代当前轻量文本描述子。
4. 将标题栏字段解析、零件名称抽取和图号映射结合起来，构建更强的结构化图纸知识检索系统。

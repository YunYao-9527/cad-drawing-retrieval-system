CAD Drawing Retrieval System

基于预训练模型微调的工程图纸检索系统

本项目面向工程图纸“结构主导、文本辅助”的检索特点，以 CLIP 视觉表征为基础，对比 YOLO 显式清洗路线与无清洗注意力式增强路线，并在候选精排阶段加入结构相似性增强和标题栏字段级 OCR 辅助，形成可运行的工程图纸检索原型系统。

系统主线

YOLO 显式清洗路线

通过检测和掩码方式尝试削弱边框、标题栏或噪声区域对检索的影响，再与结构重排和 OCR 辅助组合，验证显式清洗是否能稳定提升工程图纸检索效果。

无清洗注意力式增强路线

不删除图面内容，而是在编码阶段通过 attention-like 主体增强方式突出主体结构，再结合候选精排阶段的结构相似性增强，避免过度清洗导致的细节损失。

核心功能

  • 支持工程图纸上传检索，返回 Top-K 相似图纸、类别信息和相似度得分。
  • 使用 Qdrant 管理向量索引，支持离线建库、特征缓存和图库统计。
  • 支持 baseline、cleaning_only、masked_pooling、full_model、multimodal_text、attention_visual、attention_structure、attention_multimodal 等模式切换。
  • 提供离线评测流程，评价指标包括 Recall@K、mAP@10、nDCG@10、FT、ST 和 ANMRR。
  • 提供 Grad-CAM 可解释性分析，用于观察检索模型关注的图面区域。

关于 GitHub Pages 的说明

当前这个页面可以长期托管在 GitHub Pages 上，但 GitHub Pages 只能托管静态页面，不能持续运行 Python 后端、CLIP 模型、Qdrant 向量库或 GPU 推理服务。因此，如果需要长期可用的“真实检索系统网址”，推荐采用 GitHub 仓库 + 云服务器后端 的方式：GitHub 负责代码托管和静态入口页，后端部署到云主机、Render、Railway、Fly.io 或学校服务器，前端再访问该后端 API。

当前临时演示链接由本地服务通过 Cloudflare Tunnel 暴露，适合答辩预演或短时间展示；如需长期访问，需要把后端迁移到稳定服务器，并将本页面中的演示入口替换为正式后端地址。

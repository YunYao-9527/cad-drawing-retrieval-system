# 基于 CLIP 的工程图纸检索系统毕业论文初稿（双主线重构版，已回填结果）

## 一、重构动机

上一版实验使用的是旧测试集，图库规模只有 2400 余张，而且部分 DWG 转 PNG 图像存在黑底、黑边和截图边框干扰。这样的数据条件会同时影响视觉特征提取、结构相似性计算和 OCR 文本抽取，因此旧实验结果更适合作为阶段性探索，而不适合作为论文终稿中的正式结论。

当前项目已经切换到清理后的 `Dataset_img3` 数据集，并完成了黑底统一、视觉索引重建和 GPU 化建库链路改造。按最新脚本统计，当前数据集共有 `13880` 张 PNG 图纸，覆盖 `22` 个类别。因此，论文的方法章节和实验章节都应同步重构，而不能继续沿用旧版“五阶段串行消融”的写法。

## 二、方法路线重构

### 2.1 双主线总体思路

新的论文结构围绕两条主线展开：

1. `YOLO 清洗路线`
   先通过目标检测清理图纸中的无关干扰区域，再进行视觉检索和后续增强。

2. `无清洗注意力路线`
   不显式执行 YOLO 清洗，而是直接利用 mask-guided pooling / attention-like 机制突出主体结构，再观察其对检索性能的影响。

在这两条主线上，再分别引入两类增强模块：

1. `结构/图像相似性增强`
   通过结构描述符和重排序机制增强工程图纸的几何与布局匹配能力。

2. `OCR 文字辅助检索`
   通过标题栏 OCR、字段抽取和文本相似性计算构建多模态辅助检索能力。

### 2.2 八个实验模式

| 模式代号 | 论文名称 | 说明 |
| --- | --- | --- |
| `baseline` | Baseline | 不使用清洗、不使用注意力增强、不使用结构重排、不使用 OCR |
| `cleaning_only` | YOLO Cleaning Only | 仅使用 YOLO 清洗 |
| `masked_pooling` | YOLO Cleaning + Masked Visual | YOLO 清洗后使用 mask-guided pooling 强化主体视觉表征 |
| `full_model` | YOLO Cleaning + Structure | 在 YOLO 清洗路线中加入结构描述符和重排序 |
| `multimodal_text` | YOLO Cleaning + Structure + OCR | 在 YOLO 清洗并完成结构重排序后进一步加入 OCR 文本辅助检索 |
| `attention_visual` | Attention Visual (No Cleaning) | 不使用 YOLO 清洗，直接使用无清洗注意力式视觉建模 |
| `attention_structure` | Attention + Structure (No Cleaning) | 在无清洗注意力路线中加入结构重排序 |
| `attention_multimodal` | Attention + Structure + OCR (No Cleaning) | 在无清洗注意力路线完成结构重排序后进一步加入 OCR 文本辅助检索 |

### 2.3 注意力路线说明

这里需要在论文中如实说明：

当前代码中的“注意力路线”是可运行的 `无清洗 + mask-guided pooling / attention-like` 版本，用来回答“是否必须依赖 YOLO 清洗”这一问题。它已经可以作为论文中的一条完整实验路线。

但如果要做“严格意义上单独训练的 spatial_attention 新模型”对比，还需要专门训练一套与之匹配的权重文件。当前这一版项目中，尚未完成这一步单独重训练。因此论文中应表述为：

1. 已完成并验证的是“无清洗注意力式视觉增强路线”。
2. 尚未单独训练的是“独立 spatial_attention 新模型”。

## 三、论文章节改写建议

### 3.1 第四章方法设计建议

第四章不再按旧版“清洗 -> masked pooling -> structure rerank -> OCR”的单链条递进方式来写，而应改为如下结构：

1. `基础视觉检索框架`
   介绍 CLIP 检索主干、向量表示和检索流程。

2. `YOLO 清洗路线`
   说明通过检测框去除无关区域、生成保留掩码，并分析其对主体区域表达的影响。

3. `无清洗注意力路线`
   说明不依赖显式清洗，而是通过 mask-guided pooling 或 attention-like 机制强化结构特征。

4. `结构/图像相似性增强`
   说明结构描述符、结构重排序和前 K 候选重排机制。

5. `OCR 文字辅助检索`
   说明 OCR 文本抽取、标题栏字段抽取及多模态融合策略。

### 3.2 第五章实验设计建议

第五章改成“双主线对比实验”，而不再是旧版“五阶段串行消融”：

1. `实验环境与数据集`
   说明当前使用 `Dataset_img3`，并说明黑底统一处理和新索引构建流程。

2. `评价指标`
   使用 Recall@K、mAP@10、nDCG@10、FT、ST、ANMRR。

3. `主实验一：YOLO 清洗路线`
   对比 `baseline -> cleaning_only -> masked_pooling -> full_model -> multimodal_text`。

4. `主实验二：无清洗注意力路线`
   对比 `baseline -> attention_visual -> attention_structure -> attention_multimodal`。

5. `横向对比分析`
   重点回答两个问题：
   一是 `YOLO 清洗路线` 与 `无清洗注意力路线` 谁在视觉检索上更强；
   二是两条路线在加入 OCR 后，哪一条路线的增益更稳定。

6. `个案分析与误检分析`
   结合可视化样例、结构相似误检样例和标题栏字段匹配情况展开讨论。

### 4.4 横向比较结论

根据当前实验结果，可以直接得到以下结论：

1. `YOLO Cleaning Only` 与 `Baseline` 指标完全一致，说明在清理后的 `Dataset_img3` 上，单独引入 YOLO 清洗并没有带来额外收益。
2. 在 `YOLO 清洗路线` 中，`YOLO Cleaning + Structure` 的 Recall@1 从 0.8682 提升到 0.8795，说明结构重排序比单纯清洗更有价值。
3. 在 `YOLO 清洗路线` 中，`YOLO Cleaning + Structure + OCR` 是在 `YOLO Cleaning + Structure` 基础上继续叠加 OCR 后得到的结果；它只带来了很小的 nDCG/mAP 增益，Recall@1 反而略有下降，说明当前 OCR 融合策略还不够稳定。
4. `无清洗注意力路线` 的整体表现优于 `YOLO 清洗路线`。其中 `Attention + Structure (No Cleaning)` 在 8 个核心指标里拿到了最好的 Recall@1、Recall@5、Recall@10 和 nDCG@10，说明在当前数据条件下，无清洗注意力路线更适合作为主方法。
5. `Attention + Structure + OCR (No Cleaning)` 是在 `Attention + Structure (No Cleaning)` 基础上继续叠加 OCR 后得到的结果；它没有形成稳定增益，说明 OCR 文本辅助检索目前尚未成为性能提升的主要来源。

## 五、结果分析建议

### 5.1 可以直接写进论文的主要结论

1. 在清理后的大规模新数据集上，单纯依赖 YOLO 清洗并不能稳定提升检索性能。
2. 结构重排序对工程图纸检索仍然有效，能够提升前列候选结果的相关性。
3. 与显式清洗相比，无清洗的注意力式视觉增强路线表现更稳定，尤其在 Recall@1、Recall@5 和 Recall@10 上更优。
4. 当前 OCR 辅助检索模块尚未在新数据集上表现出稳定正增益，后续更适合将其写成“可扩展增强模块”而不是“当前最有效创新点”。

### 5.2 建议在论文里怎么表述

为了保证结论真实可信，建议不要硬写成“所有模块都稳定提升”。更合理的写法是：

1. `YOLO 清洗` 在旧数据条件下可能有帮助，但在当前经过统一清理的新数据集上，其独立贡献并不明显。
2. `结构增强` 仍然是提升检索结果排序质量的重要因素。
3. `无清洗注意力路线` 在当前实验中优于 `YOLO 清洗路线`，说明直接对主体结构进行注意力式建模可能比显式清洗更适合该任务。
4. `OCR 文字辅助检索` 的效果依赖文本质量和标题栏稳定性，当前版本增益有限，仍需进一步优化。

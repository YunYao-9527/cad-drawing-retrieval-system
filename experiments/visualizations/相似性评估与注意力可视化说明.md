# 相似性评估与注意力可视化说明

## 1. 是否存在“标准相似性评估”

严格来说，没有一种对所有任务都通用的“唯一标准相似性指标”。

不同任务通常采用不同的评估口径：

### 1.1 如果任务是“两张图片是否相似”

常用指标包括：

- 语义相似度 `Cosine Similarity`
  - 对深度特征向量计算余弦相似度
  - 适合当前 CLIP 检索框架
- 结构相似性 `SSIM`
  - 衡量亮度、对比度和局部结构的一致性
  - 更偏向图像层面对齐
- 感知相似性 `LPIPS`
  - 更适合自然图像感知差异
  - 但对工程图纸未必最合适
- 文本相似度
  - 适合标题栏、图号、零件名称等文本辅助判别

### 1.2 如果任务是“图纸检索系统好不好”

更标准的评价方式其实是检索指标：

- `Recall@K`
- `mAP`
- `nDCG`
- `FT / ST`
- `ANMRR`

因为检索任务真正关心的不是单张图之间“像不像”，而是：

- 同类图纸能不能排到前面
- 前 K 个结果里是否能召回正确候选
- 排序质量是否稳定

因此对你的毕业设计和论文来说：

- **系统级标准指标**：`Recall@K / mAP / nDCG / ANMRR`
- **样本对解释性指标**：`Cosine / Structure / Text / SSIM`

这是更合理的组合。

## 2. 本次新增的可视化方式

新增脚本：

- [`visualize_pair_similarity.py`](D:/workspace/图纸检索项目/数据集/111_test/experiments/visualize_pair_similarity.py)

它会对两张图纸生成：

1. 成对相似性热力图
2. 配套 JSON 分数报告

当前输出的指标包括：

- `embedding_cosine`
- `semantic_similarity`
- `structure_similarity`
- `text_similarity`
- `patch_alignment`
- `ssim`

其中：

- `semantic_similarity` 反映深度语义特征是否接近
- `structure_similarity` 反映几何结构布局是否接近
- `text_similarity` 反映标题栏与说明文字是否接近
- `patch_alignment` 反映 patch 级局部区域是否存在匹配
- `ssim` 反映图像层面的整体结构一致性

## 4. 论文中建议怎么写

建议将相似性评价分成两层：

### 4.1 系统性能评价

采用：

- `Recall@1`
- `Recall@5`
- `Recall@10`
- `mAP@10`
- 如需更完整可补：`nDCG / FT / ST / ANMRR`

### 4.2 个案可解释性分析

采用：

- `semantic_similarity`
- `structure_similarity`
- `text_similarity`
- `SSIM`
- patch 热力图可视化

这样写的好处是：

- 指标体系更规范
- 可视化更有说服力
- 容易同时满足“论文严谨性”和“答辩展示效果”

## 5. 下一步建议

如果继续往论文和答辩方向强化，最值得补的是：

1. 再生成 4 到 6 组“同类/异类/误检”可视化案例
2. 把热力图插入论文实验章节的定性分析部分
3. 补充 `nDCG / ANMRR`，让评价指标更接近标准 CBIR 论文写法

# 🛠️ CAD图纸检索系统 - 智能清洗补丁技术指南 (v1.1 纯YOLO版)

本指南旨在帮助开发人员将 **“YOLO 内存清洗”** 功能集成到现有的 CAD 图纸检索系统中，实现**“只存特征不存图，检索自动去干扰”**的高级功能。

> **版本说明**：此版本为纯视觉清洗版，极大提升了处理速度。

---

## 1. 📖 技术原理说明

本补丁采用 **“内存级视觉过滤”** 策略，在不修改原始磁盘文件的前提下，提升检索精度。

### 核心流程

1. **入库/检索**：读取原始图纸  **内存中清洗** (YOLO识别并覆盖表格/标题栏)  生成纯净特征  向量运算。

### 技术栈

* **视觉过滤**：使用 **YOLOv8** 自定义模型，快速识别并覆盖大面积干扰区（标题栏、BOM表、技术要求）。
* **零 IO 损耗**：所有图像处理均在 RAM 中以 NumPy 数组形式完成，不生成中间文件。

## 2. ⚙️ 环境安装配置

### 2.1 更新依赖文件 (`requirements.txt`)

由于移除了 OCR，依赖变得非常轻量。请在项目根目录的 `requirements.txt` 中添加（或保留）以下内容：

```text
ultralytics>=8.0.0
opencv-python
tqdm
# 注意：已移除 paddlepaddle 和 paddleocr

```

### 2.2 执行安装

在终端运行：

```bash
pip install -r requirements.txt

```

---

## 3. 📂 模型与路径配置

### 3.1 模型文件准备

将您训练好的 YOLO 模型文件复制到项目目录中。

* **建议路径**：`项目根目录/runs/detect/drawing_cleaner_v1/weights/best.pt`

### 3.2 配置文件修改 (`config/config.yaml`)

请在配置文件的末尾添加 `cleaning` 模块（OCR 开关已移除）：

```yaml
# ... (上文是 app, database, logging 等配置) ...

# === 新增：清洗服务配置 ===
cleaning:
  # YOLO 模型路径 (请修改为您实际存放 .pt 文件的路径)
  # 支持绝对路径，或相对于 main.py 的相对路径
  model_path: "runs/detect/drawing_cleaner_v1/weights/best.pt"

```

## 6. 🔧 常见问题排查

* **❌ 报错 `FileNotFoundError` 或 `ModuleNotFoundError: ultralytics**`
* **解决**：请检查 `requirements.txt` 是否安装，以及 `model_path` 路径是否正确。


* **⚠️ 警告 `未配置清洗模型路径**`
* **解决**：检查 `config.yaml` 中是否正确缩进了 `cleaning` 节点。
